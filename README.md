# DAF - 一款 DNF 连发工具

## 功能简介
- 多键位无冲突连发
- 使用 `vk=0xFF + 扫描码` 的游戏模式输入，尽量避免影响聊天框打字
- 仅在 DNF 游戏窗口内生效
- 支持软件启动后自动开连发、开机自启动、游戏内屏蔽 Win 键
- 支持多预设保存、快速切换预设
- 内置职业/功能扩展：旅人自动流星、关羽自动猛攻、战法自动炫纹/大炫纹、剑宗帝剑延迟、修罗自动一绝、自动宠物技能、自动奔跑、一键连招（多套方案）、自动识别配置（按参考图切换预设）


整体分成 4 层：

1. 入口层：负责启动、加载模块、初始化托盘和首屏窗口。
2. 核心层：负责连发、配置读写、按键转换和窗口识别。
3. 界面层：负责主界面、软件设置、快速切换窗口，以及各扩展功能的配置窗口。
4. 扩展层：负责职业特化逻辑、自动奔跑与自动识别配置等附加能力。

## 运行流程
`DNFAutoFire.ahk` 是项目入口，启动后会按下面的顺序工作（`/Run=…` 子进程只解析到连发与扩展逻辑所需 `#Include`，再进入 `SubProcessThread.ScriptStart()`，不加载主界面与 GDI+ 键帽等大段 GUI，以缩短多子进程拉起时间）：

1. 主进程：先加载连发与扩展共用的核心 `#Include`，再调用 `SubProcessThread.ScriptStart()`（非子进程时立即返回），随后建立主进程单实例保护、加载 GUI 等大段代码、初始化托盘并显示主窗口；带 `/Run=…` 的子进程在同一调用处按 `/Preset=…` 指定的预设进入连发或扩展逻辑并保持运行。
2. 检查并初始化 `config.ini`，确保至少存在一个默认预设。
3. 初始化托盘菜单和主窗口。
4. 读取上次使用的预设，以及软件设置中的自动启动选项。
5. 用户点击“启动连发”后，主界面会先保存当前主界面状态再隐藏到托盘；脚本最多启动两条输入运行时：`MainAutoFire` 子进程承载当前预设的所有普通连发键，`ExActionRuntime` 子进程统一承载已勾选的扩展动作；启动子进程时显式传入当前预设名，普通连发不屏蔽原始键入，专用子进程内多定时器方式发送补充输入；若当前预设没有任何可启动的输入运行时，会回到停止态并显示主界面；自动识别在主进程内注册热键，不再单独拉起子进程。

## 版本与图标
- 版本号统一维护在 `Version.ahk`：改 `APP_VERSION` 与同行的 `;@Ahk2Exe-Let AppVersion` 即可（两处数值保持一致）。
- 运行时界面直接读取编译进 exe 的 `APP_VERSION` 常量，不再依赖 `FileGetVersion` 或解析脚本注释。
- 当前版本维护地址：https://bbs.colg.cn/thread-9593722-1-1.html
- 图标文件统一放在 `lib/ui/icons/` 目录，编译时作为资源写入 exe，源码运行时托盘图标也从该目录读取。

## 目录与模块说明

### 入口文件
- `DNFAutoFire.ahk`
  - 项目总入口。
  - 统一维护版本号和编译资源入口。
  - 分两段 `#Include`：连发/扩展子进程所需核心库与 `ex/*.ahk` 在前，随后调用 `SubProcessThread.ScriptStart()`；主进程在返回后再加载 GDI+ 与主界面等 GUI 模块。
  - 初始化托盘菜单、主窗口、当前预设和自动启动逻辑。
  - 托盘仅保留“连发设置”和“退出连发”入口。
  - `SubProcessThread.ScriptStart()` 处理子进程 `/Run=…` 入口（子进程不重复执行提权段）。
  - 主进程使用互斥量做单实例保护；`#SingleInstance Off` 仍保留，用于允许 `/Run=…` 输入运行时子进程并存。

### core 核心层
- `core/AutoFire.ahk`
  - 普通连发子进程的执行循环。
  - `MainAutoFire()` 在一个专用子进程内读取当前预设的全部普通连发键，为每个键注册独立 `SetTimer`， `AutoFireSingleKeyTick()` 骨架检测物理按下状态，仅在 DNF 前台时 `SendIP`。
  - 不注册 `$*` 屏蔽热键；保留单键间隔覆盖全局间隔，并在主键专用子进程内调用 `UnlockSystemTimeLimit()` 降低定时器抖动。

- `core/Scripts.ahk`
  - 管理普通连发的开启/关闭。
  - 维护当前启用的连发按键列表。
  - 启动一个 `MainAutoFire` 主键专用子进程承载所有普通连发键；若有扩展功能开启，则再启动一个 `ExActionRuntime` 子进程统一承载扩展动作，启动参数会带上当前预设名，主进程和普通连发子进程都不屏蔽原始键入。
  - 停止连发时显式关闭子进程并确认退出；运行态切换只走主界面的显式启动/停止流程，不再额外挂一个定时看门狗轮询子进程状态。
  - 启动和停止扩展功能，切换托盘图标，处理预设切换。

- `core/Config.ahk`
  - 统一读写 `config.ini`。
  - 管理“软件设置”和“预设设置”两类数据；可选的 `PresetOrder` 键用于主界面预设列表的自定义排序。
  - 负责首次运行时创建默认配置。

- `core/KeyConvert.ahk`
  - 负责按键名和输入格式转换。
  - 当前普通连发和多数扩展都通过 `vkFFscXX` 形式发送按键，适配 DNF 的游戏输入场景。

- `core/ComboPreset.ahk`
  - 一键连招多套方案的序列化/解析（主进程与子进程共用），每套方案保存触发键、循环、屏蔽原键、出手延迟和技能序列；录入按键后保留采集到的 AHK 键名，运行时再转扫描码和游戏输入 token；技能序列用 `|` 分隔技能，用 `:` 分隔技能键和间隔；方案数量不设固定上限，空方案以 `blank` 保存并在读取时还原为空方案对象，`ComboProfiles=` 表示无方案，并提供 INI 导入/导出格式。

- `scripts/migrate_combo_profiles.ahk`
  - 一键连招旧配置迁移脚本；主程序不兼容读取旧技能序列格式，需要迁移旧 `ComboProfiles` 时运行 `& 'C:\Program Files\AutoHotkey\v2\AutoHotkey.exe' scripts\migrate_combo_profiles.ahk`，可加 `/DryRun` 只查看需要迁移的预设，也可传入指定 ini 路径。

- `ex/ExActionRuntime.ahk`
  - 扩展输入动作运行时：按预设构建规则并统一承载旅人、关羽、宠物技能、战法、剑宗、修罗、自动奔跑和一键连招。
  - 旅人、战法、剑宗、修罗、宠物技能、关羽和一键连招的触发键先按扫描码汇总到统一热键分发器，同键只注册一次 AHK 热键再依次分发到各功能；同一扫描码按住期间重复触发的 Down 会被分发器忽略，直到收到 Up 后才允许下一次触发；按住补发类扩展由分发器维护按下/抬起状态，定时器只负责按间隔补发。修罗触发键仍会在普通主连发运行时中专门避让，避免和修罗扩展互相叠加；关羽复用一键连招的“出手延迟 + 序列发送”模型，使用技能键触发、出手延迟后发送一发猛攻，松开会取消未触发的猛攻；一键连招以负定时器链顺序 `SendIP` 发送技能键，支持按方案设置出手延迟、按住循环，并可按方案选择是否屏蔽触发键原键透传，运行中触发其他连招方案会打断当前连招并立即启动新方案。

- `core/SendIP.ahk`
  - 封装一次完整的按下/抬起发送动作，是普通连发、扩展与一键连招的基础发送接口；使用 `SendEvent` 发送 `vkFFscXX` 游戏输入。
  - 一键连招与关羽序列发送路径显式使用 12ms 按下保持时间，与 0.2.8 等效；其他功能仍按各自配置或默认值调用。
  - 内部使用按脚本路径命名的全局 Mutex 串行化跨子进程发送，避免 `MainAutoFire` 与 `ExActionRuntime` 同时 `SendEvent` 时互相挤掉输入。

- `core/AutoPresets.ahk`
  - 自动识别配置：搜图（`ImageSearch`）匹配城镇与各配置下的技能栏小图（每配置可多张），决定切换到哪套预设；参考图圆角外侧使用纯白遮罩，搜图时将该纯白遮罩当作任意颜色；连发启动或 `Esc`/附加热键触发后延迟 500ms 开始识别，每 500ms 一轮、最多 60 轮（城镇识别成功可提前结束）；每轮先识别城镇，识别成功则立即停止本次后续重试，未识别则识别技能栏并进入下一轮，仅技能匹配到对应配置时才切换。热键仍只在「已开启自动识别」、连发会话进行中、DNF 窗口前台时生效。
  - 自动识别内部使用会话编号和识别序列编号隔离开始等待与重试定时器；手动触发会取消当前待执行识别并重新开始，识别过程中若 DNF 不在前台会暂停重试，回到前台后继续，停止或重启连发后旧定时器回调会自动失效。

### gui 界面层
- `gui/main/Main.ahk`
  - 主界面。
  - 提供键盘面板、预设列表、快速切换热键、扩展功能入口。
  - 「当前配置名称」使用只读输入框显示，外观与下方间隔、热键等字段统一。
  - 使用系统主题控件，让主界面按钮外观与扩展设置窗口保持一致；「其他功能」各扩展开关为 GDI+ 滑动样式，采用更接近 Win11 的轻量 hover/pressed/短过渡反馈，逻辑仍记在隐藏勾选状态上，写入预设的时机与原先主界面一致（切换预设、关闭主窗口、启动连发等会触发保存）。
  - 主界面键帽的点击命中已统一收口到键帽本体；无论点到键帽底图、文字还是 Shift 左侧辅助箭头，都会按对应键名立即切换状态，避免叠层文本命中后丢失键名导致的“点不动”假象。
  - 所有标准 GUI 窗口统一支持轻量“伪失焦”：当焦点停在文本框、热键框或列表等控件上时，点击窗口空白处会把焦点切到离屏占位控件，达到接近网页点击空白退出编辑态的效果，同时不干扰现有 hover 命中层。
  - 采用列表驱动的预设管理方式，支持单击切换、在配置列表上右键新建/重命名/克隆/删除、左键拖动排序（顺序保存在 `设置\PresetOrder`，排序不会切换当前预设）。
  - 控件创建已统一走 `lib/ui` 包装层，便于后续调整整体外观和布局。
  - 主界面「按键设置」「配置设置」「其他功能」分区标题在原布局基础上统一下移 6px；「配置设置」「其他功能」标题的右边线与右侧按钮区对齐。
  - 主界面「其他功能」开关行距为 28px。
  - 主界面「其他功能」左列放置旅人、关羽、战法、修罗，右列从帝国剑术开始排列。
  - 键盘区标准 104 键布局由 `lib/ui/Layout.ahk` 中的 `MainKeyLayoutData` 计算，`Main.ahk` 的 `MainBuildKeyboardPanel()` 生成键帽；主界面布局参数集中在 `gui/main/MainLayout.ahk`。

- `gui/MainText.ahk`、`gui/exText.ahk`、`gui/AutoPresetsText.ahk`
  - 集中维护主界面/设置/快速切换、扩展功能窗口、自动识别窗口的界面文案。

- `lib/ui/*.ahk`
  - 界面主题、布局和控件包装层。
  - 统一窗口背景、字体、键帽颜色、按钮、标签、列表、输入框等基础控件创建入口。
  - 当前正文界面字体统一为 `Microsoft YaHei UI`，主界面键帽文字继续使用 `Segoe UI`，兼顾中文可读性与键帽观感一致性。
  - 主界面分区标题使用透明背景文本，允许标题占位延伸到右侧按钮区而不遮住按钮边缘。
  - EX 页标题/小节标题的帮助按钮与底部保存按钮对齐规则统一收敛在这一层；帮助按钮固定放回右上角，并按内容区右缘对齐。
  - 键帽采用 GDI+ 自绘扁平样式（与参考项目 DNFAutoFire 一致），启用为蓝色高亮、关闭为灰底描边；字号按键位类型区分（字母 `s12`、修饰键 `s10`、方向键 `s14`），方向键箭头保持原有样式，并单独使用 4 倍超采样配合更平滑的高质量缩小插值方式提升清晰度。
  - 主界面 GDI+ 自绘开关同样使用 4 倍超采样，并在缩小时采用更平滑的高质量插值，保证边缘更自然。
  - 提供 `UiRect`、基于控件矩形累计内容边界的 `UiContentLayout`、通用技能键列表编辑器，以及可复用的 ListBox 拖动排序行为。
  - 统一提供窗口 hover 快照与空白点击伪失焦能力；空白点击时只在窗口背景命中后切焦，不影响现有控件 hover/pressed 反馈。
  - 主界面和扩展设置窗口优先调用这一层，后续可在不改业务逻辑的情况下替换控件外观。
- `lib/ui/icons/*.ico`
  - 托盘和编译资源使用的图标文件。

- `gui/Setting.ahk`
  - 软件设置窗口，包含自动启动、开机启动、屏蔽 Win 键、子进程异常日志、关闭按钮最小化到托盘等通用选项。
  - 包含自动启动连发、开机启动、屏蔽 Win 键等全局选项。

- `gui/QuickSwitch.ahk`
  - 快速切换窗口。
  - 通过热键呼出，支持上下选择预设，回车或空格直接切换并启动。

- `gui/AutoPresets/*.ahk`
  - 自动识别设置窗口与屏幕区域框选（技能栏、城镇判定）；窗口按顶部触发设置、中部城镇参考图、下部配置列表与技能图列表（每配置可绑定多张技能图，默认名称角色1/2/3…，双击改名）及技能参考预览分区排布；参考图文件落在 `assets/preset-recognition/skills/<配置名>/`；框选窗口与截取参考图使用一致的圆角裁剪，且框选窗口只在打开期间注册窗口消息监听，关闭时同步卸载。
  - 自动识别窗口布局常量集中在 `gui/AutoPresets/AutoPresetsLayout.ahk`，窗口宽高按控件布局自动推导。

- `gui/ex/*.ahk`
  - 各扩展功能的独立设置窗口。
  - 负责采集触发键、技能键、出手延迟等参数并保存到当前预设。
  - 基础控件统一调用 `lib/ui` 包装层，重复的技能键列表编辑窗口共用同一套布局构建器；旅人、关羽、战法、宠物技能键列表，以及一键连招方案/技能序列支持拖动排序；一键连招方案可单独勾选「屏蔽原键」，双击方案可设置出手延迟，默认 20ms，方案列表不显示 0ms 和 20ms，允许清空全部方案，并支持导入/导出当前窗口内的全部方案。
  - EX 窗口布局入口集中在 `gui/ex/ExLayout.ahk`，设置/快速切换等主界面子窗口的布局入口集中在 `gui/main/MainLayout.ahk`。
  - EX 窗口布局统一遵守一条内容区右边线：页头标题、问号帮助按钮、右侧表单、整行按钮与底部保存按钮都按同一个 `contentRight` 对齐；窗口边框留白由 `ExLayout` 统一提供，新增页面不要再手写 `+28`、`+32` 之类的问号偏移。

## EX 窗口布局规则
- 统一从 `gui/ex/ExLayout.ahk` 取 EX 页公共边距与页头参数，不在单个页面里重新定义边框留白。
- 每个 EX 页面先确定自己的 `contentRight`，它表示“内容区右边界”，不是窗口宽度；右对齐控件、帮助按钮和底部保存按钮都只认这条线。
- 页头统一调用 `UiExPageTitle(...)`，传入 `contentRight`；不要再把标题宽度当成帮助按钮定位基准，也不要在页面里写 `contentRight + 28`、`contentRight + 32` 这类补偿值。
- 底部保存按钮统一走 `UiExSaveButtonRect(...)`，左边从 `ExLayout.MarginLeft()` 起，右边贴 `contentRight`。
- 技能键列表型 EX 优先复用 `UiSkillKeyEditor(...)`（目标键位用 `UiPressKeyEdit` 点击录入，不再单独放「设置按键」按钮）；普通表单型 EX 的单键位设置同样使用 `UiPressKeyEdit`；各页面复用同一套左边距、右边线和控件宽度规则，保证切换时边缘对齐一致。

### ex 扩展层
- `ex/ExActionRuntime.ahk`
  - 统一扩展运行时，按策略承载多种触发模型：旅人/战法炫纹为“多触发键按住后按主键间隔补发一个键”，战法大炫纹为“复用战法触发键按住检测但每次按下只触发一次”，宠物技能为“多触发键热键边沿发一次”，关羽为“独立运行项复用一键连招序列模型：技能键触发、出手延迟后发一发猛攻、松开取消未触发动作”，剑宗为“单触发键按住并延迟后持续补发同键”，修罗自动一绝为“单触发键同时驱动两组不同频率的补发”，一键连招为“单触发键通过定时器链执行多技能序列，可按方案设置出手延迟并屏蔽触发键原键透传，运行时 1～19ms 的出手延迟按 20ms 执行，0ms 立即执行，运行中触发其他连招方案会打断当前连招并立即启动新方案”，自动奔跑为方向键热键动作（可配置触发延迟与按住关闭键；按住时仅跳过额外补发）。旅人、战法、剑宗、修罗、宠物技能、关羽和一键连招共享统一热键分发器，同一扫描码只注册一次 AHK 热键，重复触发键会分发给对应功能；若同键下任意一键连招方案启用「屏蔽原键」，该键按屏蔽原键方式注册；修罗触发键仍由 `MainAutoFire` 专门避让。
  - 运行时统一做 DNF 前台判断、按住类触发键热键状态维护、宠物技能等边沿类热键触发、焦点丢失状态清理和 `vkFFscXX` 发送；普通主键连发仍保持在独立 `MainAutoFire` 子进程。

### 新增 EX 开发提示
- EX 功能统一收敛到 `ex/ExActionRuntime.ahk`，新增功能优先注册到统一运行时，不再拆新的常驻子进程脚本。
- 主界面入口集中在 `gui/Main.ahk`，文案集中在 `gui/MainText.ahk` / `gui/exText.ahk`，设置窗口放在 `gui/ex/*.ahk`，预设保存/读取由 `core/Scripts.ahk` 和对应设置窗口协作。
- 常见“触发键列表 + 目标键 + 延迟”类页面优先复用 `UiSkillKeyEditor(...)`；单键录入复用 `UiPressKeyEdit(...)`；需要调整键列表顺序时复用 `UiListBoxDragSort_Attach(...)`。
- 运行时优先复用 `ExAction_AddRepeatRule(...)`、`ExAction_AddEdgeRule(...)`、`ExAction_AddDelayOnceRule(...)`、`ExAction_AddDelayRepeatRule(...)` 这些规则构建器；现有 `policy` 表达不了新语义时，再在规则 tick 中补最小必要策略。
- 新增功能通常只需要补设置窗口、主界面开关/入口、文案、预设字段读取保存，以及 `ExAction_BuildRules(presetName)` 中的规则注册。

### lib 通用库
- `lib/MultipleThread.ahk`
  - 用“子进程脚本”方式承载普通连发与扩展功能。
  - 子进程入口通过分发表执行 `MainAutoFire` 或 `ExActionRuntime`，并使用 `/Preset=…` 明确运行预设，不再保留旧的单键连发参数回落入口；各子进程的输入发送仍通过 `SendIP` 的全局 Mutex 串行化。
  - 子进程启动异常默认不写入日志；在「软件设置」中勾选「子进程异常错误日志」后，会静默写入 `subprocess-error.log`，避免弹窗打断主流程，同时保留排查线索。

- `lib/Keys.ahk`
  - 提供所有可选键位的键位总表。

- 其余库
  - `RunWithAdministrator.ahk`：管理员权限启动辅助。
  - `GetPressKey.ahk`：阻塞式按键采集（如 EX 技能列表「添加」）；默认等待按键时会通过 `ShowTip` 提示「输入按键..」，但不会切换到游戏窗口；调用方也可关闭 `ShowTip` 并自行在控件上显示录入状态（一键连招「添加技能」按钮会临时显示「输入按键..」）。
  - `lib/ui/PressKeyEdit.ahk`：可点击只读框的单键采集；录入时显示「输入按键..」，Esc 清空，失焦恢复原值。
  - `Time.ahk`、`Log.ahk`：通用工具支持。

## 当前功能拆解

### 1. 普通连发
- 在主界面点亮一个或多个按键后，这些键会加入当前预设。
- 在「配置设置」区域右侧可设置「全局连发间隔」（毫秒，全局保存）：默认 `20`，有效范围为 `1` 或更大的整数，作为对应按键定时器的触发周期；切换预设不会改变该值。
- 在键帽（标签或键图）上**右键**可打开菜单，通过同一个入口为**该键**设置独立连发参数，弹窗内分别填写「连发间隔」和「按键延迟」；连发间隔需为 `1` 或更大的整数毫秒，按键延迟需为 `0` 或更大的整数毫秒，恢复默认会同时清除该键的独立间隔和独立延迟。有独立参数的键在键帽**右上角**会显示一条小横条（与键帽 GDI+ 绘制中的提示条样式一致）。单键间隔保存在当前预设的 `AutoFireKeyIntervals` 字段，单键按键延迟保存在 `AutoFireKeyDelays` 字段（均为 `键名:毫秒` 用 `|` 拼接）；子进程内若某键在表中有记录，则使用该键独立值，否则间隔使用全局值、按键延迟使用默认 `8ms`。
- 启动连发后，主界面会隐藏，脚本会启动一个 `MainAutoFire` 子进程；该进程按启动参数中的预设名读取所有普通连发键，为每个键注册独立定时器，不安装普通连发屏蔽热键，并在 DNF 前台时发送补充输入。
- 只要按住对应键，就会持续发送同键输入。
- 左键点击托盘图标、右键选择“连发设置”，或在快速切换窗口里点击“停止连发”时，会走同一套逻辑：停止连发并显示主界面。

### 2. 预设管理
- 每个预设都保存一套独立的连发键、单键连发间隔、扩展开关和扩展参数；全局连发间隔保存在软件设置中，所有预设共用。
- 支持默认预设、快速切换、单击切换、新建空配置、重命名、克隆和删除。
- 主界面修改键位或扩展选项后，会先停留在界面状态；连发运行时主界面一定隐藏在托盘，显示主界面时一定处于停止连发状态，切换配置、启动连发或关闭主窗口前，会自动保存到当前预设。
- 主界面按 `Esc` 或点右上角关闭时都会保存当前预设；默认直接退出程序，若在「软件设置」勾选「关闭按钮最小化到托盘」，则会保存当前预设并隐藏主界面，托盘菜单仍可重新打开或退出。
- 扩展功能窗口保持独立保存模型：点击保存、按 `Esc` 或点右上角关闭时，都会立即保存到当前预设。
- 上次成功加载的预设会写入 `config.ini`，下次启动自动恢复并完整加载。

### 3. DNF 窗口限定
- 主键连发专用子进程只在 DNF 窗口组前台时发送补充输入，不额外注册屏蔽热键。
- 扩展功能和窗口热键逻辑都会检查 DNF 窗口组。
- 当前只按精确窗口标题匹配游戏窗口：`地下城与勇士：创新世纪` 与 `次元对决`；新增兼容目标时，将其窗口标题加入 `core/Scripts.ahk` 的 `GAME_WINDOW_TITLES`。
- `ShowTip`（如「连发已启动」、自动识别切预设）延迟约 50ms 后先在 DNF 客户区右下角显示；显示前按精确窗口标题尝试 `WinActivate` 回到游戏，约 1 秒后消失。「连发已启动」不附带配置名。
- 自动识别首次等待、失焦暂停后的恢复检查、每轮间隔均为 500ms，单次识别序列最多 60 轮；城镇识别成功则立即停止后续轮次，城镇未识别则继续识别技能栏并进入下一轮（`core/AutoPresets.ahk`）。

### 4. 快速切换
- 默认热键是 `Alt + \``。
- 可在游戏内快速呼出预设窗口，直接切换配置并启动或停止连发；切换时会先完整停止当前连发，再按目标预设重新进入运行态；停止后会关闭快速切换窗口并显示主界面。

### 5. 附加扩展
- 扩展功能全部挂在预设下保存，可以按职业分别配置（「自动识别」总开关 `AutoPresetsEnabled` 为全局项，保存在 `[设置]`）。
- 主连发启动时，若有任意扩展被勾选，会启动一个 `ExActionRuntime` 子进程并在其中按预设启用对应扩展逻辑。
- 修罗自动一绝：设置一个触发键、一个平X键和三个波动剑键后，按住触发键会同时触发四个键的连发；平X跟随全局连发间隔，三个波动剑按全局连发间隔的三倍发送；若触发键本身也开启了主连发，运行时会自动跳过该触发键的普通连发，避免和修罗扩展互相叠加。
- 通过主界面右侧「自动识别」打开设置；在设置窗口顶部勾选「连发运行时启用本功能」即写入全局项 `AutoPresetsEnabled`，同一区域可录入附加触发键；启动连发后会自动进入一次识别序列，连发运行且已开启时也可在 DNF 前台按 `Esc`/附加热键重新触发；命中另一套预设的技能参考图时，主进程会停止当前连发并以新预设重新进入运行态；识别过程中若切出 DNF 会暂停，回到前台后继续；设置窗口下部显示配置列表、技能图列表与参考预览，最底部为「保存」。

## 配置文件说明
项目配置统一保存在根目录的 `config.ini` 中，主要分为两类：

- `[设置]`
  - 保存全局选项，如上次预设、自动启动、开机启动、Win 键屏蔽、快速切换热键；以及自动识别的开关（`AutoPresetsEnabled`）、框选区域、附加热键、城镇参考图路径相关键名（见 `core/AutoPresets.ahk` 与 `core/Config.ahk` 中的 `AutoPreset*` 项）。

- `[预设:名称]`
  - 保存某个预设的连发键列表、扩展开关，以及扩展各自的参数。
